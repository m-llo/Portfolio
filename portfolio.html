Tradify.IO
(Group Project)

Motivated by the recent financial revolution happening in US markets, we wanted to develop a tool that empowers investors of all types, levels, and interest. With Tradify, savvy investors who want to peer into the world of stocks, cryptocurrency, and foreign exchange markets, can access our platform to get up to the minute pricing data and insight in three financial markets in one tool, instead of having to use a separate tool for each.

Application Features
  • WHEN a user enters the site
  • THEY are presented with featured stocks and currencies
  • WHEN a user enters the symbol for a stock, crypto, or foreign currency in to the search bar and clicks search
  • THEY are presented with their desired stock, crypto, or foreign currency and it's current price
  • WHEN a user clicks the save to favorites icon
  • THEY can enter the symbol for a stock, crypto, or foreign currency in to the search bar
  • WHEN a user clicks add
  • THEIR desired stock, crypto, or foreign currency is saved to local storage and displayed in the Favorites section

Script Features
  1. HTTP requests (using the Fetch API)
  2. Event listener (onclick)
  3. Object, function, array, for loop and if condition
  4. Dynamically create new DOM elements
  5. Local storage

Weather Dashboard

My challenge was to build a weather dashboard that will run in the browser and feature dynamically updated HTML and CSS.

Application Features
  • WHEN a user searches for a city
  • THEY are presented with current and future conditions for that city and that city is added to the search history
  • WHEN a user views current weather conditions for that city
  • THEY are presented with the current tempeartue, humidty, wind speed, and UV Index.
  • WHEN a user views the UV index
  • THEY are presented with a color that indicates whether the conditions are favorable, moderate, or severe
  • WHEN a user views future weather conditions for that city
  • THEY are presented with a 5-day forecast that displays the date, an icon representation of weather conditions, the temperature, and the humidity
  • WHEN a user clicks on a city in the search history
  • THEY are again presented with current and future conditions for that city
  • WHEN a user opens the weather dashboard
  • THEY arepresented with the last searched city forecast

Script Features
  1. HTTP requests (using the Fetch API)
  2. Event listener (onclick)
  3. Object, function, array, for loop and if condition
  4. Dynamically create new DOM elements
  5. Local storage

CURATIO!
(Group Project)

Curatio allows hobbyist to create custom playlist of their favorite YouTube how-to and other related videos. After creating a hobby playlist, users can view individual videos and write and save notes with timestamps, best practices, and other important information.

Application Features
  • When a user creates a unique, password protected user account
  • THEY can save thier hobbies by name.
  • WHEN a user saves a hobby they can then search for Youtube videos related to that hobby or interest
  • THEY can save as many videos as they like to a unique hobby playlist
  • WHEN a user views thier playlist they can select any video to view
  • THEY can create and save notes related to that specific video

Features
  1. Full Stack server-based application using Node.js
  2. MVC format with unique models and routes.
  3. Express.js for router functionality.
  4. Handlebars.js templates to render HTML elements.
  5. MySQL and Seqelize for database construction and function.
  6. Node-fetch for server based external api fetch requests.
  7. Bcrypt.js for password and data encryption.
  8. Node-Mailer for for account creation confirmation emails.
